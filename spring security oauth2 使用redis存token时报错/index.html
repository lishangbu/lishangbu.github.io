Spring Security Oauth2 使用RedisTokenStore出错

发表于 2018-05-15 | 更新于 2018-11-09 | 评论数：

问题

我将我的Spring Cloud项目版本升到Finchley.M9后，集成了spring security oauth2的三件套,采用RedisTokenStore来存储token。项目正常启动后，请求token时报错。

贴出报错信息:

看到这个报错,我的第一反应就是版本出现了冲突，但是我的依赖的都是spring-boot-starter中的默认版本，然后在谷歌中找了一下，发现了这个：
https://github.com/spring-projects/spring-security-oauth/issues/1230#event-1352041281。
果然是版本问题，spring-data-redis。然后我按照网页中的解决方法“spring-date-redis”改为2.0.1.RELEASE和”jedis“为2.9.0（显式声明），结果还是报一样的错，后来这个页面我又看了看，下面的人也是一堆同样的问题。怪我看issues没要仔细:joy::joy::joy:

解决方案

出现这个问题的原因主要是spring-data-redis 但是我不想降版本，所以我用了一个极端的方式解决的。spring-data-redis 2.0版本中set(String,String)被弃用了，要使用RedisConnection.stringCommands().set(…)，所有我自定义一个RedisTokenStore，代码和RedisTokenStore一样，只是把所有conn.set(…)都换成conn..stringCommands().set(…)，测试后方法可行。代码如下：

至此问题就当解决:joy::joy::joy:

更优雅的解决方案

会出现这个问题主要是因为Finchley.M9的spring-security-oauth2依赖的redis版本过低，升级到2.3.3版本就可以解决这个问题。
